
Fleet Platform- 로봇 관제 시스템에 대해 얘기해 봅시다.  로봇을 판매하거나 제작할 때 로봇의 위치를 모니터링 하는 시스템에 같이 보여지게도 하고, 따로 존재하기도 하는 관제 시스템은 Modern Robot SW 에서는 주로 Cloud 위에 존재하고 로봇은 실시간으로 모니터링 할 정보들을 보내고, Cloud 관제 시스템은 제어를 위한 Mission(혹은 Task)를 주게 됩니다.
introduction 에서 보았던 전체 그림에서 아래와 같은 위치를 차지하고 있습니다.
![[Excalidraw/FleetManagementExplained01.excalidraw|100%]]

관제의 기본적인 도식은 아래 그림처럼 간단하게 정의할 수 있습니다. 서버에서 Mission을 주고 로봇은 그 미션을 수행합니다. 미션을 수행하면서 서버에게 작업의 진행 과정을 전달하는 것을 Report 라고 합니다. 이 Report 데이터를 적절하게 사용자에게 전달 하기 위해 여러가지 Visualization tool 을 통해 데이터를 정제해서 제공하는 데 이런 일련의 행위를 Monitoring이라고 합니다. 
![[Excalidraw/FleetManagementExplained02.excalidraw|100%]]

User 즉, 고객이 누구 인가에 따라서 기존 고객이 가지고 있는 Monitoring 도구들과 Integration 할 수도 있고, 독립적으로 모니터링 툴을 제공하기도 합니다. 

이것이 일반적인 관제의 가장 기초적인 ABC 이지만, 이것을 가지고 Fleet Platform 이라고 하지는 않습니다. 
기술한 데로, 고객의 업종과 형태에 따라서 제공해야 하는 것들이 많습니다. 특히 여러가지 서비스들과 같이 하나의 로봇 서비스로 만들어져야 하는 경우에는 API를 제공해 줘야 한다 든 지, 로봇 지도를 공유해야 한다 든 지 등의 고려해야 할 사항들이 많습니다.
